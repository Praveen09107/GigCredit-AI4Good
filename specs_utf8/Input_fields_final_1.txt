GLOBAL PROCESSING RULES
????????????????????????????????????????????????????????????????????????????????

ABSENT DATA HIERARCHY:
  LEVEL 1 -- Step not submitted:
    All features for that step = 0.0
    Confidence contribution = 0.0
    Score pulled fully toward NEUTRAL_BASELINE = 0.50

  LEVEL 2 -- Step submitted, document rejected by validation:
    Same treatment as Level 1

  LEVEL 3 -- Step submitted, OCR partial failure:
    VERIFIED fields ? use normally
    PARTIAL fields  ? use with 50% confidence weight
    FAILED fields   ? 0.0

  LEVEL 4 -- Feature mathematically undefined:
    income_recovery_speed (no gaps) ? 1.0 (no gaps = perfect)
    max_to_avg_ratio (avg = 0)      ? 1.0 (default neutral)
    balance_growth_slope (1 month)  ? 0.0 (insufficient history)

MINIMUM SCORING REQUIREMENTS (refuse to score if any fail):
  ? Primary bank statement absent (P1 confidence < 0.35)
  ? Aadhaar not verified
  ? Face similarity < 0.70

NaN / INFINITY RULE:
  Any NaN or Infinity result ? replace with 0.0 immediately
  Flag in meta.computation_warnings list
  NEVER propagate NaN into float arrays

DIVISION BY ZERO:
  All division operations check denominator > 0 before computing
  If denominator = 0 ? apply Level 4 default values

BLOCKED FIELD RULE:
  Social Category (Udyam field #5) must NEVER be passed to
  any feature vector, model input, or scoring function.
  Extract only for display_only = true profile record.
