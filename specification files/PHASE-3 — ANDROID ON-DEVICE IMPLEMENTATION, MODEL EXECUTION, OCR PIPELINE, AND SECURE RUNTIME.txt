PHASE-3 — ANDROID ON-DEVICE IMPLEMENTATION, MODEL EXECUTION, OCR PIPELINE, AND SECURE RUNTIME


OBJECTIVE


Define the exact execution workflow inside the Android application.


This phase explains:


• How models are loaded and decrypted
• How OCR, AI detection, and validation run together
• How temporary storage works
• How feature engineering runs on-device
• How final scoring happens


This phase is the CORE runtime of the system.


---


SECTION-1 — APP INITIALIZATION (APP START)


Execution begins when user opens the app.


FLOW:


App Launch
↓
Initialize Secure Environment
↓
Run Security Checks
↓
Load Models
↓
Prepare Runtime


---


SECURITY CHECKS (MANDATORY)


1. Root Detection
   Detect if device is rooted


2. Debugger Detection
   Check if app is being debugged


3. Hook Detection
   Detect frameworks like Frida/Xposed


4. App Signature Verification
   Ensure APK not tampered


IF ANY FAILS:


→ Disable model execution
→ Show error


---


SECTION-2 — ANDROID KEYSTORE + MODEL DECRYPTION


IMPORTANT CONCEPT


Models are stored as:


model.enc (encrypted)


They must be decrypted ONLY at runtime.


---


KEYSTORE FLOW


App First Launch
↓
Generate AES Key
↓
Store inside Android Keystore


---


MODEL LOAD FLOW


Load model.enc
↓
Fetch key from Keystore
↓
Decrypt in memory (RAM)
↓
Initialize TensorFlow Lite Interpreter


IMPORTANT RULES:


• Decrypted model NEVER stored on disk
• Exists only in RAM
• Cleared after use


---

FLOW:

MODEL:

↓

↓
Classification output

SECTION-7 — FACE VERIFICATION (STEP-2 ONLY)


FLOW:


Document Image
↓
MediaPipe Face Detector
↓
Crop Face
↓
MobileFaceNet
↓
Generate embedding


Selfie Image
↓
Same pipeline


↓


Cosine similarity


---


DECISION:


«0.75 → verified
0.60–0.75 → review
<0.60 → reject»


---


SECTION-8 — VALIDATION ENGINE


3 LAYERS:


---


1️⃣ INDIVIDUAL VALIDATION


Examples:


PAN format → regex
IFSC → regex


---


2️⃣ CROSS-INTERNAL VALIDATION


Example:


Bank statement fields must match:


name
account number
IFSC


---


3️⃣ CROSS-STEP VALIDATION


Example:


PAN name == Bank name
ITR PAN == Step-2 PAN


---


IF ANY FAILS:


→ reject step
→ user must re-upload
