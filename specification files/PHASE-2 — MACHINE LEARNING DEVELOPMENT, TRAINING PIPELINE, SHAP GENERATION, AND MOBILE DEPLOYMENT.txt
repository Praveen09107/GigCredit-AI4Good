OBJECTIVE


Define the complete machine learning lifecycle:


• Feature pipeline creation
• Model training (P1–P6)
• SHAP explainability generation
• Model optimization for mobile
• Conversion to TensorFlow Lite
• Model encryption for secure on-device deployment


This phase ensures the models are:


• Accurate
• Lightweight
• Explainable
• Secure
• Mobile-compatible

SECTION-1 — DATASET DESIGN


INPUT DATA SOURCES


Training dataset must simulate real gig worker data:


• Bank transaction history (6–12 months)
• UPI transaction history
• Utility bill payments
• Insurance data
• ITR declared income
• Work type (driver, vendor, freelancer, etc.)


Each user record should contain:


raw_user_data = {
transactions: [...],
upi_transactions: [...],
utility_payments: [...],
itr_data: {...},
work_type: "...",
insurance_data: {...}
}

SECTION-2 — FEATURE ENGINEERING PIPELINE (TRAINING TIME)


IMPORTANT:


This pipeline MUST EXACTLY MATCH on-device implementation.


Any mismatch → wrong predictions.


---


STEP-1: DATA CLEANING


• Remove invalid rows
• Normalize date formats
• Handle missing values
• Remove duplicates


---


STEP-2: TRANSACTION TAGGING ENGINE


Each transaction must be categorized.


Tags:


GIG_INCOME
SALARY
EMI_DEBIT
UTILITY_PAYMENT
SAVINGS
P2P_TRANSFER
CASH_WITHDRAWAL


Example:


"HDFC LOAN EMI" → EMI_DEBIT
"TNEB BILL" → UTILITY_PAYMENT
"SWIGGY Payout" → GIG_INCOME


---


STEP-3: MONTHLY AGGREGATION


For each month:


monthly_credit
monthly_debit
monthly_balance
monthly_emi
monthly_utility


---


STEP-4: FEATURE COMPUTATION


Total features: ~95


Examples:


avg_monthly_income
income_variance
emi_to_income_ratio
utility_payment_ratio
bounce_count_ratio
savings_ratio
upi_usage_ratio


All features must be:


• Float32
• Normalized [0–1]


---


STEP-5: FEATURE NORMALIZATION


Use:


min-max normalization OR domain-based normalization


Example:


income_norm = income / (anchor × 2.5)


---


STEP-6: FINAL FEATURE VECTOR


feature_vector = [f1, f2, f3, ..., f95]


---


SECTION-3 — MODEL TRAINING


You will train MULTIPLE models.


---


PILLAR MODELS


P1 → Income Stability
P2 → Payment Discipline
P3 → Debt Management
P4 → Savings Behaviour
P6 → Financial Resilience


NOTE:


P5 is rule-based → NOT ML


---


MODEL TYPE (RECOMMENDED)


Use:


• LightGBM OR XGBoost


Reason:

TRAINING PIPELINE


feature_vector
↓
Split dataset (train/test)
↓
Train each pillar model
↓
Evaluate (RMSE / accuracy)
↓
Save trained models


---


META-LEARNER MODEL


Combines pillar outputs.


Input:


[P1, P2, P3, P4, P6]


Output:


Final credit score (300–900)


---


SECTION-4 — SHAP EXPLAINABILITY GENERATION


IMPORTANT CLARIFICATION:


SHAP is ONLY used during training.


---


PROCESS


trained_model
↓
TreeExplainer (SHAP)
↓
Compute SHAP values
↓
Aggregate across dataset
↓
Compute mean contribution


---


STORE OUTPUT


feature_shap_coefficients[95]
feature_population_means[95]


---


WHY THIS IS IMPORTANT


On device:


You DO NOT run SHAP


Instead:


impact = coefficient × (value − mean)


---

Convert models to mobile format.

SECTION-6 — MODEL ENCRYPTION
